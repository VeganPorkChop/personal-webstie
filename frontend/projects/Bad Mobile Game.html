Pathfinder Puzzle

A compact Android sliding-tile puzzle built with Jetpack Compose. Rearrange numbered tiles to re-form a winding path with touch-friendly controls and offline play.

This page summarizes the current repository state, build instructions, architecture guidance, and release notes.

Project Overview

Pathfinder Puzzle is intended to be a lightweight Android puzzle game built with Jetpack Compose. The app presents a sliding-tile board where players rearrange numbered tiles to re-form a contiguous, winding path. The repository currently contains Android module scaffolding and Gradle configuration required to build the app.

Goals

  • Provide a lightweight, offline-friendly puzzle game.
  • Keep the UI focused on touch-friendly controls.
  • Make the project easy to extend with new board sizes, color themes, and rule variations.

Current Repository State

  • Android app module: Configured for Compose and Material 3.
  • Manifest: AndroidManifest.xml defines the application entry point.
  • Gradle configuration: Version catalog values live in gradle.properties, while plugin and repository configuration are in settings.gradle.

Setup & Build

This project is an Android application module. The instructions below cover the baseline requirements for building and running the app.

Prerequisites

  • Android Studio Flamingo or newer.
  • Android SDK with compileSdk 34 installed.
  • JDK 17 (Android Studio bundles a compatible JDK).

Initial setup

  • Open the repository in Android Studio.
  • Allow Gradle to sync and download dependencies.
  • Confirm the Compose compiler version is set via compose_ui_version in gradle.properties.

Common Gradle commands

Run these from the repository root:

./gradlew assembleDebug
./gradlew test
./gradlew lint

Running on a device

  • Create or start an Android emulator (API 24+ recommended).
  • Select the device in Android Studio.
  • Click Run to install and launch the app.

Build artifacts

  • Debug APK: app/build/outputs/apk/debug/
  • Release APK: app/build/outputs/apk/release/

Note: Signing configuration is not committed yet; see Release Checklist.

Architecture Notes

This repository currently contains the Gradle configuration and Android manifest for the Pathfinder Puzzle app. Use the guidance below when building out the Kotlin source structure.

Module layout

app/
  src/main/
    AndroidManifest.xml

When adding source code, prefer the standard Android layout:

app/
  src/main/java/com/example/pathfinder/
    MainActivity.kt
    ui/
      theme/
      components/
    data/
      repositories/
      models/
    domain/
      usecases/

Recommended layers

  • UI layer: Compose screens, state holders, and UI-specific models.
  • Domain layer: Puzzle rules, shuffle logic, move validation.
  • Data layer: Persistence for best times and settings.

State management

When implementing gameplay, keep state in a single source of truth (e.g., a ViewModel). Compose should render state without duplicating it.

Testing ideas

  • Unit test puzzle shuffling for solvability.
  • Unit test move validation logic.
  • UI test for tap-to-slide interactions and timer behavior.

Configuration Reference

This project uses standard Gradle configuration files for Android builds.

gradle.properties

Key properties are stored in gradle.properties:

  • android_gradle_plugin_version: Android Gradle plugin version.
  • kotlin_version: Kotlin compiler version.
  • compose_ui_version: Compose compiler extension version.

When upgrading Compose, update the Compose BOM in app/build.gradle and keep this property in sync.

settings.gradle

Plugin versions and repository configuration are defined in settings.gradle:

  • pluginManagement configures the Android and Kotlin plugins.
  • dependencyResolutionManagement ensures dependencies resolve from Google and Maven Central.

app/build.gradle

Key app settings:

  • compileSdk 34 and targetSdk 34.
  • minSdk 24 for API compatibility.
  • Compose and Material 3 dependencies.

Tips

  • Keep Java and Kotlin at version 17 unless you have a specific reason to change.
  • Use the Gradle wrapper (./gradlew) for consistent builds.

Gameplay Rules

Pathfinder Puzzle is a sliding-tile puzzle. The goal is to restore the numbered tiles to a contiguous path pattern as quickly as possible.

Core rules

  • The board contains numbered tiles and a single empty space.
  • A tile can move into the empty space if it is orthogonally adjacent.
  • The timer starts on the first move.
  • The puzzle is solved when the tiles form the intended path layout.

Player actions

  • Tap a tile next to the empty space to slide it.
  • New game should shuffle the board into a solvable configuration.
  • Reset timer should restart the clock without reshuffling.

Scoring & persistence

  • Track the fastest completion time on-device.
  • Store best times locally so they persist between sessions.

Extension ideas

If you want to evolve the gameplay, consider:

  • Variable board sizes (3x3, 4x4, 5x5).
  • Alternate victory conditions (e.g., ordered numbers vs. path layout).
  • Accessibility options (high-contrast colors, larger tiles).

Release Checklist

Use this checklist when preparing a release build.

Pre-flight

  • Update versionCode and versionName in app/build.gradle.
  • Verify minSdk, targetSdk, and compileSdk values.
  • Run unit tests and lint checks.

Build release APK

./gradlew assembleRelease

Artifacts will appear in app/build/outputs/apk/release/.

Signing

Release signing configuration is not yet stored in the repository. When you add it:

  • Use a keystore stored outside the repo (or via CI secrets).
  • Add a signingConfigs block to app/build.gradle.
  • Avoid committing credentials; instead rely on Gradle properties or environment variables.

Post-build validation

  • Install the APK on a device and run a full play-through.
  • Verify that best-time persistence works across restarts.
  • Confirm any analytics or crash reporting configurations.

Troubleshooting

Use this section to diagnose common build and run issues.

Gradle sync failures

  • Ensure you are using the Gradle wrapper (./gradlew).
  • Confirm network access to Google Maven and Maven Central.
  • Clear Gradle caches if dependency downloads are corrupted:
./gradlew --stop
rm -rf ~/.gradle/caches

Missing Compose compiler version

The build expects a compose_ui_version property. If you see a Missing compose_ui_version error, ensure gradle.properties contains it and that the file is not ignored.

Device launch issues

  • Ensure your emulator/device targets API 24+.
  • Uninstall previous debug builds if installation fails due to signature conflicts.

Still stuck?

Gather the following information for debugging:

  • Android Studio version.
  • Full Gradle stacktrace (./gradlew assembleDebug --stacktrace).
  • Device or emulator API level.
