A full Scrabble engine — premium board, exhaustive legal-move generation over the tournament word list, complete cross-word scoring, and a neural network that learned move value from self-play.

Project Overview

Scrabble AI is a complete implementation of the board game: a 15×15 board with double/triple letter and word premiums, the full TWL06 tournament dictionary (178,690 words), a bag and rack manager, and an engine that can enumerate every legal move from a position. On top of that sits ScrabbleNet, a neural network trained by self-play to judge which of those moves is actually best.

Pieces of the engine

  • GADDAG move generator — the Appel & Jacobson algorithm, backed by a GADDAG data structure, finds all legal plays.
  • Full scoring — letter and word premiums, every cross-word formed, and the 50-point bingo bonus.
  • ScrabbleNet — a three-stream network (board CNN + rack MLP + move MLP) that estimates each move's value.

Try the engine

Deal a random rack (or type your own — ? is a blank), then hit Find best moves. The engine ranks its legal plays by score; hover to preview a play on the board, click to make it. Watch for the orange BINGO plays that use all seven tiles for a +50 bonus.

About this version: the browser engine reproduces the real one with a trie-based generator (the same anchor + cross-check idea as the GADDAG) over a bundled TWL06 subset (all words of length 2–7), and it ports the real scoring code verbatim. Ranking here is by raw score; the full project's ScrabbleNet adds learned positional judgement on top.

Move generation

Generating legal Scrabble moves is the hard part. The algorithm rests on three ideas:

  • Anchors — any empty square touching an existing tile. On an empty board the center star is the only anchor. Words must pass through an anchor to connect to the board.
  • Cross-checks — for each anchor, precompute which letters are even legal there, given the perpendicular word they would form. A letter that makes an invalid down-word is rejected before the main search ever tries it.
  • Dictionary walk — starting from each anchor, walk the word structure letter by letter, placing rack tiles and matching existing board tiles, pruning the instant the prefix leaves the dictionary.

The real project uses a GADDAG, which cleverly encodes words so the search can grow leftward from an anchor and then rightward in one traversal. The browser demo uses a plain prefix trie with explicit left/right handling — simpler, same result.

Scoring

Every play is scored exactly as in tournament Scrabble, and this logic is shared byte-for-byte between the real project and the demo:

  • Letter premiums (DL, TL) multiply a tile's value — but only on the turn it is first placed.
  • Word premiums (DW, TW) multiply the whole word; two triple-words stack multiplicatively.
  • Cross-words — every perpendicular word your new tiles create is scored too.
  • Bingo — using all seven rack tiles in one turn adds 50 points.
score = Σ (letter × letter_premium) × word_premium
        + Σ cross_words
        + (50 if all 7 tiles used)
