Connect Four AI

A convolutional Deep-Q network that taught itself Connect Four through self-play — no opening books, no hand-written strategy, just millions of games and a reward signal.

Scroll down to play a game against it right in the browser.

Project Overview

Connect Four AI is a reinforcement-learning agent that plays the classic 6×7 drop game. It is built in PyTorch as a Deep-Q Network (DQN): a convolutional network estimates, for the current board, how good it is to drop a disc in each of the seven columns. The agent starts knowing only the rules and improves by playing itself.

Highlights

  • Convolutional Q-network — reads the board as an image so a pattern learned in one place transfers everywhere.
  • Self-play training — after an initial phase against a "smart-random" opponent, the agent trains against copies of itself.
  • Reward shaping — small hints for building threats, forks, and center control accelerate learning without overriding the real win signal.
  • Target network + replay buffer — the usual DQN stabilizers keep learning from diverging.

How the AI works

Board encoding — a 3-channel image

The 6×7 board is fed to the network as a (3, 6, 7) tensor:

  • Channel 0: 1 where the agent has a disc.
  • Channel 1: 1 where the opponent has a disc.
  • Channel 2: 1 for columns that still have room.

Keeping the two players in separate channels lets the network see "my pieces" and "their pieces" as distinct features. A flip_state trick swaps the channels on the opponent's turn, so the network only ever has to answer one question: what's best for the player in channel 0?

Choosing a move

At play time the network outputs seven Q-values; illegal (full) columns are masked to −∞ and the agent drops in the best remaining column. During training it uses ε-greedy exploration, annealing ε from 1.0 down to 0.05 over 8,000 games so it keeps exploring well into self-play.

Training

Reward shaping

The real reward is simple: +1 for a win, 0 for a draw. But sparse rewards learn slowly, so the environment adds small shaped hints that stay well below the win signal:

center preference        +0.01 · (3 - |col - 3|)/3
build a three-in-a-row   +0.10 each
build a two              +0.02 each
fork (two threes)        +0.15
leave opponent a three   -0.12 each
opponent can win next    -0.50
pointless vertical stack -0.05 · (run - 1)

These same ideas drive the evaluation of the playable browser opponent above.

Stability tricks

  • Replay buffer of 100k transitions, sampled in batches of 1,024.
  • Target network copied from the online network every 500 games.
  • Random phase of 3,000 games vs a smart-random opponent before self-play, so early learning sees legal, varied positions.

Notes

The training loop, network, and self-play harness live in agent.py, model.py, and game.py. Saved checkpoints (model.pth, plus earlier attempts) capture the weights at different points in training.
