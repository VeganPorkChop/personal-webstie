DFT Bomb Finder

Tell an explosion from an earthquake using nothing but a seismogram — the same signal-processing problem nuclear-test monitoring agencies solve, scaled down to run on a laptop.

Project Overview

DFT Bomb Finder (bombfindr_ai) detects, discriminates, and geolocates seismic events from real-world seismograph data. The hard question isn't "did the ground shake" — it's "was that an explosion (a bomb, a nuclear test, a quarry blast) or a natural earthquake?" The answer is written in the frequencies of the wave, which a Discrete Fourier Transform pulls apart.

What it does

  • Detects the P-wave arrival with an STA/LTA trigger.
  • Discriminates explosion vs earthquake from frequency-domain features.
  • Locates the source by inverting arrival times across stations.

Try it

Simulate an explosion or an earthquake. The tool picks the P arrival, runs a DFT on the signal window, extracts the physical discriminants, and classifies the event — then triangulates a source from three stations.

About this version: the real project downloads live waveforms from global seismic networks and classifies them with a trained RandomForest. The web page can't reach those archives, so it feeds a synthetic seismogram through the exact same feature math and the project's transparent physics classifier (heuristic_score), ported verbatim. The DFT, the discriminants, and the decision rule are the real thing.

The pipeline

seismograms  →  P-pick (STA/LTA)  →  features (DFT)  →  discriminate  →  locate
                                                        (explosion?)     (lat, lon)
  • data — find azimuth-balanced stations and download filtered waveforms (FDSN networks).
  • picking — STA/LTA short-term/long-term energy ratio flags the P arrival.
  • features — a DFT of the P window yields the discriminants below.
  • discriminate — a RandomForest (with a physics-heuristic fallback) outputs P(explosion).
  • locate — a travel-time model inverts arrival times to a source location.

The physics: why frequency tells the tale

Explosions and earthquakes radiate energy differently, and the Fourier transform makes the difference measurable:

  • P/S amplitude ratio — explosions push out strong compressional (P) energy and weak shear (S); earthquakes are the opposite.
  • HF/LF ratio & spectral centroid — explosions are richer in high frequencies and sit higher in the spectrum.
  • Rise time & complexity — explosions are impulsive with short codas; earthquakes are emergent with long, complex tails.
  • Kurtosis — a measure of how "spiky" (impulsive) the waveform is.

The transparent classifier adds up simple physics votes:

score  = 0.30 if P/S ratio     > 2.0
       + 0.25 if HF/LF ratio    > 1.0
       + 0.20 if centroid       > 2.5 Hz
       + 0.15 if rise time      < 3.0 s
       + 0.10 if complexity     < 1.0
P(explosion) = min(score, 1.0)

In the demo, the highlighted feature rows are exactly the votes that fired.

Notes

The pipeline modules are data.py, picking.py, features.py, discriminate.py, and locate.py, orchestrated by scripts in scripts/ and served by an interactive Flask map in webapp/. It builds on the original firstattempt.py locator. Data comes from the USGS earthquake catalog and FDSN waveform archives (not NASA — that's Mars).

This page is part of a portfolio; the interactive discriminator runs the real feature math on a synthetic signal so it needs no network access.
